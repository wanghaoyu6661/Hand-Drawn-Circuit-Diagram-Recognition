# Hand-Drawn-Circuit-Diagram-Recognition

Deep learning-based recognition and structural parsing pipeline for hand-drawn circuit diagrams.

> **Current status**
> - ✅ Core pipeline reconstructed and runnable locally (YAML-driven).
> - ✅ Final JSON to SPICE netlist export is now integrated into `scripts/run_all.sh`.

---

## 1. Project Overview

This project provides an end-to-end pipeline for recognizing hand-drawn circuit diagrams and generating:

- intermediate outputs for each stage;
- final structured JSON (`*.final.json`);
- visualization images (`*_final.png`);
- SPICE netlists (`*.spice`).

The main pipeline entry is:

```bash
bash scripts/run_all.sh
```

## 2. Quick Start

Clone the repository:

```bash
git clone git@github.com:wanghaoyu6661/Hand-Drawn-Circuit-Diagram-Recognition.git
cd Hand-Drawn-Circuit-Diagram-Recognition
```

Initialize the `mmpose` submodule:

```bash
git submodule update --init --depth 1
```

If you need a full recursive clone instead:

```bash
git submodule update --init --recursive
```

---

## 3. Path Configuration

Recommended workflow:

```bash
cp configs/paths.example.yaml configs/paths.local.yaml
```

Then edit `configs/paths.local.yaml` and replace `__PROJECT_ROOT__` with your actual absolute project path.

The default AutoDL-style path used in `configs/paths.yaml` is:

```text
/root/autodl-tmp/Hand-Drawn-Circuit-Diagram-Recognition
```

New output path added for SPICE export:

```text
paths.spice_netlists: <project_root>/outputs/run1/spice_netlists
```

This directory is created automatically by the netlist export script.

## 4. Main Outputs

After `bash scripts/run_all.sh`, the main outputs are typically:

```text
outputs/run1/
├── final_result/

│   └── json/           # *.final.json
└── spice_netlists/     # autogenerated *.spice netlists
```

## 5. SPICE Netlist Export

The final step of the pipeline runs:

```bash
python src/pipeline/build_spice_netlists.py
```

Behavior:

- it reads `outputs/run1/final_result/json/*.final.json`;
- it writes one `.spice` file per diagram into `outputs/run1/spice_netlists/`;
- it supports every component category listed in `configs/yolo_data.yaml`;
- well-defined primitives such as resistors, capacitors, inductors, voltage/current sources, diodes, BJTs, and MOSFETs are exported as native SPICE elements when the required terminals are available;
- higher-level or ambiguous symbols (for example logic gates, ICs, relays, sockets, and other abstract device symbols) are exported as placeholder `X...` subcircuits together with autogenerated stub `.SUBCKT` definitions, so the produced netlist remains syntactically complete.

This means the exported netlist is suitable as a structured machine-readable starting point. For accurate circuit simulation of high-level blocks, you can later replace the autogenerated placeholder subcircuits with your own device libraries or hand-authored models.

---

## 6. Notes

- `build_spice_netlists.py` uses OCR text matched to components when possible, so values such as `10kΩ`, `100mH`, or `9V` can be transferred into the generated netlist.
- Nets touching `gnd` / `vss` symbols are normalized to SPICE node `0`.
- The script is self-contained and reads the same YAML path configuration as the rest of the pipeline.
